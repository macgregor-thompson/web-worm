Asynchronous JavaScript

6/16/2020

Asynchronicity is the backbone of how modern web applications work...yet JavaScript is:

Single Threaded - executes one line at a time Synchronous in nature - executes code in the order it appears

JavaScript Under the Hood:

1) Thread of Execution 2) Memory 3) Call Stack

So what happens if we have a task that is accessing Twitter that takes a long time? What if we have code we want to run in the meantime? When will everything execute?

What the heck? That doesn't make sense...I thought JavaScript executed code in the order it appears...

JavaScript is not enough...we need add some more components to our three from above:

1) Web Browser - JavaScript plus more: Network Requests (Fetch, XHR) DOM (document) Local Storage & Session Storage (localStorage, sessionStorage) Timer (setTimeout) Others browser features include dev tools, console, sockets 2) Promises 1) Begin some brower work in the bacgound 2) Immediately returns a placeholder object (promise) in JavaScript 3) Event Loop 4) Task Queue 5) Micro Task Queue
